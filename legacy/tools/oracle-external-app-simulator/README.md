# Oracle External App Simulator

This folder helps you validate the Oracle common-ground design with one external app before you allow real third-party applications to write into the hub.

## Safe test path

1. Configure Oracle Free Tier connection details in `.env`.
2. Start Garmetix.
3. Sign in as Admin.
4. Open `http://localhost:3000/oracle-sync`.
5. Click **Test Oracle**.
6. Click **Repair Storage**.
7. Click **External App Test**.
8. Confirm that a `Customer` event from `ExternalAppSmokeTest` appears in the inbound queue.
9. Apply or reject it manually.

This test writes one event into Oracle with `SOURCE_APPLICATION = ExternalAppSmokeTest`, which proves that Garmetix can pull an event generated by another connected app.

## Manual SQL option

Use `insert-sample-customer-event.sql` from SQL Developer, SQLcl, or another Oracle tool after replacing placeholders:

- `__TENANT_ID__`
- `__EVENT_ID__`
- `__CUSTOMER_ID__`
- `__COMPANY_ID__`

Garmetix will pick this up on the next **Pull** when `ORACLE_SYNC_DIRECTION=PullFromOracle` or `Bidirectional`.

## Production rule

Do not enable auto-apply for external applications until you define entity ownership. Customers, vendors, products, categories, stores, companies, and employees are shared master entities. Billing, purchase, stock, GST, accounting, and ledgers remain blocked by default.
